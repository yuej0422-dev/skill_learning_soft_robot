# 状态空间技能学习

## 1. 定位

状态空间技能学习是当前最优先落地的下一步。它建立在两个已有基础之上：

- 成熟的气动软体机械臂底层控制器；
- 提前训练好的状态空间 Koopman encoder。

该阶段不直接学习从观测到气压的端到端控制，而是学习高层技能策略，让技能策略输出期望 TCP 位姿、速度、子目标或轨迹 chunk，再交给底层控制器转换为气压。

## 2. 控制接口

底层控制器接口保持不变：

\[
(x_{tcp}^{des}, \dot{x}_{tcp}^{des}) \rightarrow pressure
\]

其中：

- \(x_{tcp}^{des}\)：期望 TCP 位姿；
- \(\dot{x}_{tcp}^{des}\)：期望 TCP 速度；
- \(pressure\)：气动机械臂的气压控制量。

技能策略负责生成底层控制器可执行的期望量：

\[
\pi_{skill}(s_t, goal) \rightarrow (x_{tcp}^{des}, \dot{x}_{tcp}^{des})
\]

也可以输出更长时间尺度的轨迹 chunk：

\[
\pi_{skill}(s_t, goal) \rightarrow
\{x_{tcp,k}^{des}, \dot{x}_{tcp,k}^{des}\}_{k=0}^{H-1}
\]

## 3. 状态表示

最小状态可以使用：

\[
s_t=[x_{tcp}, \dot{x}_{tcp}]
\]

其中包含 TCP 位姿和 TCP 速度。

如果任务中存在物体交互，则状态可以扩展为：

\[
s_t=[x_{tcp}, \dot{x}_{tcp}, o_t, \dot{o}_t]
\]

其中 \(o_t\) 表示物体位姿或任务相关对象状态。

若使用提前训练好的状态空间 Koopman encoder：

\[
z_t^S=E_{koop}^{state}(s_t)
\]

则技能策略可以使用：

\[
\pi_{skill}(z_t^S, goal)
\]

或：

\[
\pi_{skill}([s_t,z_t^S], goal)
\]

状态空间 Koopman encoder 只服务于该状态技能学习环节，不与后续视觉语言空间 Koopman 强制共享。

## 4. 技能输出形式

推荐优先级如下：

1. **输出期望 TCP 子目标**：实现简单，最稳，便于复用已有底层控制器。
2. **输出期望 TCP 轨迹 chunk**：适合连续操作和短时动作规划。
3. **输出 delta TCP action chunk**：更接近策略控制，但对数据质量和稳定性要求更高。
4. **不建议首版直接输出气压**：这会绕开成熟底层控制器，增加学习难度和安全风险。

## 5. 训练方法

首版建议采用监督模仿学习：

- state-only behavior cloning；
- GRU 轨迹模型；
- ACT / action chunking；
- teacher forcing 训练轨迹 chunk；
- rollout 后收集失败或偏差片段，再用 DAgger 式数据修正。

如果使用 Koopman error，可以构造目标条件误差：

\[
e_t=z_t^S-z_g^S
\]

然后训练：

\[
\pi_{skill}(e_t, z_g^S) \rightarrow (x_{tcp}^{des}, \dot{x}_{tcp}^{des})
\]

该方案适合目标明确、终态可定义的操作任务。

## 6. 数据采集

每个时间步建议记录：

\[
\mathcal D_t=
\{s_t,z_t^S,goal_t,x_{tcp}^{des},\dot{x}_{tcp}^{des},pressure_t,\tau_t\}
\]

如果包含物体操作，还应记录：

- 物体位姿；
- 目标区域；
- 接触或抓取阶段标签；
- 成功/失败标记；
- episode id 和 reset 边界。

## 7. 验证指标

状态空间技能学习的首版评估应关注：

- 技能成功率；
- 最终 TCP 位置误差；
- 最终 TCP 姿态误差；
- 完成时间；
- 轨迹平滑度；
- 气压变化是否平稳；
- 不同初始状态下的泛化能力。

该阶段的目标是先获得稳定可展示的技能能力，而不是立即解决视觉语言端到端控制。
