# 总的技术流程

## 1. 前置基础

当前系统已经具备成熟的气动软体机械臂底层控制器：

```text
期望 TCP 位姿 + 期望 TCP 速度
        ↓
底层控制器
        ↓
气压控制量
```

因此后续技术路线不再重复设计底层控制器，而是围绕更高层的技能学习、视觉语言微调、去动捕状态恢复和轻量 VLA 展开。

同时，状态空间 Koopman encoder 会提前训练好，用于状态空间技能学习等需要状态动力学表征的环节。该 Koopman encoder 只属于状态空间环节，不代表全系统共享的唯一 embedding。

## 2. 总路线

整体路线按由稳到难、由状态到视觉语言的顺序推进：

```text
状态空间技能学习
        ↓
动捕 + RGB-D 同时配备下的 VLA 微调
        ↓
去动捕的 RGB-D 视觉本体状态恢复
        ↓
RLT + 视觉语言空间 Koopman 表征压缩
        ↓
基于 Goal / Koopman Error 的软体领域轻量 VLA
```

对应目标为：

1. **状态空间技能学习**：在可靠动捕状态和成熟底层控制器基础上，尽快做出可展示、可复现的技能学习 demo。
2. **动捕 + RGB-D + VLA 微调**：动捕继续提供可靠状态，RGB-D 和语言进入 VLA 数据流，复用成熟 VLA SFT / 部署流程。
3. **去动捕视觉本体状态恢复**：尝试从主视角 RGB-D 多帧观测和历史控制中恢复本体状态，逐步替代部署阶段的动捕输入。
4. **RLT + 视觉语言空间 Koopman**：在训练 RLT 或紧凑 RL 状态时，引入视觉语言空间的 Koopman encoder，用于压缩 VLM/VLA embedding 和建模局部演化。
5. **轻量软体 VLA**：围绕 goal predictor、任务误差和偏差控制，构建面向一类软体操作任务的数据高效轻量 VLA。

## 3. Koopman 使用原则

本路线不假设存在一个全局共享 Koopman embedding。

不同环节可以有不同的 Koopman encoder：

- **状态空间 Koopman encoder**：输入动捕或估计的本体状态，用于状态空间控制、状态技能学习或状态动力学表征。
- **视觉状态恢复 Koopman**：如果在 RGB-D 去动捕阶段使用 Koopman，对应的是视觉状态恢复环节的局部动力学约束。
- **视觉语言空间 Koopman encoder**：在 RLT/VLA 阶段压缩 VLM/VLA hidden states 或 token embedding，用于任务阶段、动作后果或误差空间建模。

这些 Koopman 表征可以在接口层互相提供监督或辅助信息，但不强制共享同一个 latent 空间。

## 4. 推荐推进顺序

第一步先完成状态空间技能学习，因为它最依赖现有成熟能力，风险最低。

第二步同步采集 RGB-D、语言、动捕状态和控制数据，为 VLA 微调和后续去动捕提供数据资产。

第三步在不破坏原有状态控制链路的前提下，引入 VLA 作为上层目标、技能或动作 chunk 生成器。

第四步训练 RGB-D 视觉本体状态恢复模型，逐步用视觉估计状态替代动捕状态。

第五步在 VLA/RLT 后训练中引入视觉语言空间 Koopman encoder 和偏差控制机制，提升泛化性和数据效率。
